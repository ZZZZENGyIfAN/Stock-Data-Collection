# Stock-Data-Collection

一个用于学习和实践的 **股票证券数据采集项目**。  
项目的目标是从公开财经数据源采集股票与指数相关数据，并将结果整理后保存到数据库，方便后续做数据分析、量化研究、报表统计或自动化更新。

这个项目不只是单个爬虫脚本，而是按照“可维护、可扩展”的思路拆分成多个模块，覆盖了从数据抓取、清洗、入库，到日志记录、定时任务运行的一整套基本流程。

---

## 项目是做什么的

这个项目主要用于采集和管理以下几类证券数据：

- **证券基本信息**  
  包括股票代码、名称、交易所、行业、上市日期等。

- **股票日行情数据**  
  包括开盘价、最高价、最低价、收盘价、成交量、成交额等常见市场数据。

- **财务报表数据**  
  包括资产负债表、利润表、现金流量表等公司基本面数据。

- **指数历史行情**  
  用于跟踪大盘或代表性指数的走势情况。

- **股票相关新闻**  
  用于辅助分析市场信息、舆情变化和个股动态。

项目最终会把这些数据写入数据库，形成一个可持续更新的本地证券数据仓库。

这个项目适合以下场景：

---

## 项目的整体工作方式

整个项目的核心思路可以概括为：

1. 从不同的数据源抓取不同类型的证券数据  
2. 对接口返回的数据进行解析和清洗  
3. 将结构化结果写入 MySQL 数据库  
4. 记录运行日志，便于排查问题  
5. 通过脚本或定时任务实现自动更新

也就是说，这个项目的重点不只是“能抓下来”，而是尽量把抓取流程整理成一个能长期维护的工程结构。

## 文件夹说明

### `collectors/`
这个文件夹存放**各类数据采集器**。

每个采集器负责一种数据类型，例如：

- 证券基本信息采集
- 股票日行情采集
- 财务数据采集
- 指数历史行情采集
- 股票新闻采集

这里可以理解为整个项目的“核心业务区”，真正负责与外部数据源交互、解析数据并组织入库逻辑。

---

### `config/`
这个文件夹存放**项目配置**。

通常包括：

- 数据库连接信息
- 请求超时设置
- 重试次数
- 请求头（如 User-Agent）
- 日志配置
- 其他全局参数

把配置集中放在这里，方便统一修改，也避免把账号、端口、超时等信息写死在业务代码里。

---

### `db/`
这个文件夹存放**数据库相关代码和 SQL 文件**。

主要负责两部分内容：

- Python 里如何连接数据库、执行 SQL、批量写入数据
- 项目需要创建哪些表，以及表结构如何设计

它是项目的数据落地层，负责把采集结果真正保存下来。

---

### `logs/`
这个文件夹用于存放**运行日志**。

项目执行过程中，通常会把以下信息记录到日志里：

- 当前正在抓取哪个模块
- 接口请求是否成功
- 抓到了多少页、多少条数据
- 数据保存是否成功
- 程序报错信息

这个目录的意义是帮助开发者排查问题，也方便后续查看历史运行情况。

---

### `scripts/`
这个文件夹存放**入口脚本**。

这些脚本一般不负责具体抓取逻辑，而是负责“组织执行流程”，例如：

- 初始化数据库
- 执行一次完整采集
- 执行一次日常增量更新

可以把它理解成项目的“启动入口”或“调度层”。

---

### `utils/`
这个文件夹存放**通用工具代码**。

这里通常放一些可以被多个模块重复使用的辅助功能，比如：

- 日志工具
- 公共函数
- 解密脚本
- 数据处理小工具

它的作用是减少重复代码，让采集器和脚本文件本身更清晰。

---

### `.venv/`
这是本地 Python **虚拟环境目录**。

它里面保存的是：

- 当前项目使用的 Python 环境
- 安装的第三方依赖包
- pip 相关工具文件

这个目录主要服务于本地开发运行，一般不属于项目核心业务代码的一部分。

## 项目特点

这个项目相比“单文件爬虫脚本”，更强调下面这些点：

- **模块化**  
  不同数据类型拆成不同采集器，职责更清晰。

- **配置集中管理**  
  便于修改数据库、请求参数和运行环境。

- **数据库落地**  
  数据不是只打印在终端，而是正式保存到表中。

- **日志可追踪**  
  出现问题时可以从日志里定位原因。

- **支持日常更新**  
  可以作为一个长期运行的数据采集小系统，而不是一次性脚本。

---

## 项目运行的大致流程

这个项目通常会按下面的思路运行：

- 先初始化数据库表
- 再执行某一个采集模块，验证是否能抓到数据
- 然后逐步运行其它采集模块
- 最后通过统一入口脚本组织全量或日常更新任务

在实际使用时，最常见的方式通常是：

- 第一次运行时做一次完整初始化
- 后续按天执行更新脚本
- 持续把新增数据补充进数据库

## 总结

`stock_data_collector` 是一个围绕**证券数据采集、清洗、入库和更新**搭建的 Python 项目。  
它适合用来练习一个相对完整的爬虫型数据工程项目，而不仅仅是写几个临时抓取脚本。

如果你希望从“会抓网页数据”进一步走向“能搭一个有结构、能维护、能持续运行的小型数据系统”，这个项目就是为这个目标设计的。
